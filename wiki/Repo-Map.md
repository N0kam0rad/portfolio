# Repo Map

The portfolio is a static GitHub Pages site served from the repository root.

## Root pages

- `index.html` — landing page with hero, selected work, about, and contact footer
- `portfolio.html` — full project grid with app cards and gated demo buttons
- `blog.html` — build-log index
- `tools.html` — AI and builder tools page
- `ideas.html` — idea backlog / concept garden
- `contact.html` — contact and consulting page

## App demos

Single-file HTML app demos live in:

```text
apps/
```

App files should be static, self-contained, and safe for GitHub Pages.

## Product specs

Mini product specs live in:

```text
specs/
```

Specs should explain the product clearly, not just describe the UI.

## Build logs

Long-form posts live in:

```text
posts/
```

Build logs should show process, screenshots, prompting, decisions, mistakes, and lessons.

## Assets

Images and visual assets live in:

```text
assets/
```

Expected asset types:

- Project thumbnails
- Blog screenshots
- Cast references
- Character expression packs
- App UI screenshots
- Demo stills

## Important note

Some older handoff notes may say the deployed root is `site/`. That is stale for the current repo. The live site is currently rooted at the repository root.
