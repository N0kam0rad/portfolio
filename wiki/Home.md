The deployed site currently serves from the repo root, not a `site/` subfolder.

Key paths:

- `index.html` — landing page
- `portfolio.html` — full project grid
- `blog.html` — build log index
- `posts/` — long-form build logs
- `specs/` — product spec pages
- `apps/` — single-file HTML app demos
- `assets/` — thumbnails, screenshots, character refs, and visual assets

## Working principle

The portfolio should make the work feel real, usable, and product-minded. Every project should answer:

1. What is it?
2. Who is it for?
3. What problem did it solve?
4. What did AI help with?
5. What did human judgment still control?
6. What would be built next?
