# Project Roster

This page tracks the portfolio projects and what each one needs to feel complete.

## Status rule

Nothing should be described as shipped unless it is truly usable and public-ready.

Use these labels:

- **Prototype** — clickable, usable demo exists.
- **In progress** — UI or webapp is being built, but it is not usable yet.
- **Draft** — concept, story system, spec, or early visual direction only.
- **In production** — active creative production is underway, but final output is not shipped.

## Display rule

Projects with real thumbnails or live visual previews should display first on both:

- `index.html`
- `portfolio.html`

Placeholder cards should move below screenshot-backed projects until they receive thumbnails. A thumbnail is proof of direction, not proof of shipment.

## App-card standard

Every project card should include:

- Project name
- Honest status
- Real thumbnail or live preview
- One-line description
- What exists now
- What still needs to be built
- Spec link
- Prototype link only when a usable prototype actually exists
