## The thesis

AI does not replace taste, direction, product thinking, or the ability to know when something is giving “template with a wig on.” It just helps me move from **idea → prototype → product story** before the spark evaporates.

```text
fuzzy idea       → product framing
voice notes      → specs + build logs
AI assist        → prototype + polish
human judgment   → story, taste, usability, shipping
```

---

## What this is

This repo powers my public portfolio: a single-page site plus product specs and build logs for the apps, workflows, and creative systems I’m building with AI.

The site is designed to show:

- what I built
- why I built it
- what problem it solves
- what tools did the work
- what broke along the way
- what I learned as a product-minded AI builder

## Repo structure

```text
portfolio/
├── index.html          # live single-page portfolio
├── assets/             # thumbnails, screenshots, and visual assets
├── specs/              # product spec pages
├── posts/              # build-log articles
├── apps/               # lightweight runnable app references
├── .nojekyll           # GitHub Pages helper
└── README.md           # this file
```

---

## How I add a new project

Every new product follows the same pattern:

1. Save the app/source in `apps/`.
2. Create a product spec in `specs/`.
3. Add a project card to `index.html`.
4. Add screenshots or thumbnails to `assets/`.
5. Write a build-log post when the process is interesting.
6. Push to GitHub Pages.

---

## Design notes

The visual system is intentionally bold and tactile:

- warm paper canvas
- near-black ink
- hard borders
- offset shadows
- oversized type
- bright section colors
- project cards with status labels
- open build logs that explain how the thing got made

This is not just a gallery. It is a proof-of-work machine.
