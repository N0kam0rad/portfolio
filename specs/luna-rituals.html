Luna's Rituals

A routine-and-reward app for kids: complete your morning and bedtime rituals, earn points, unlock daily puzzles, and save toward real prizes — with a parent approval step.

Built · in testingSingle-file appLocal-firstGamified
What it is

The 10-second version

A playful mobile app where a child taps through their morning/bedtime routine, earns ⭐ points per step, reveals a hidden daily puzzle as they go, and saves points toward a prize wishlist. Cashing out a prize pings the parent to approve — points-as-allowance, gamified.

Why I built it

The spark

Getting a kid through a routine is a daily battle, and most chore apps feel like spreadsheets for adults. I wanted something a child would want to open — rewards that feel earned, not nagged.

The problem it solved for me

Motivation that sticks

A routine tool built around delight instead of compliance — streaks, confetti, and a mystery to uncover — so the habit runs itself instead of running on reminders.

The problem it solves for a user

Every parent's morning

Parents everywhere need a low-friction way to build kids' habits and teach saving. This turns the routine into a game the child drives, with the parent holding the approval — useful far beyond one household.

How it works

Under the hood

A single self-contained HTML app — no install, no accounts, runs from a home-screen shortcut. Each routine step reveals a piece of a hidden picture; finishing the routine unlocks a quiz for bonus points. Streaks, a prize wishlist with progress bars and "Shop" links, and a parent-approved cash-out. All progress saves on the device.

Where it's going · how it monetizes

The business model

  • Freemium app — core routines free; premium puzzle packs, themes, and multi-child profiles as a small subscription.
  • Affiliate — the prize wishlist links to retailers (the "Shop" buttons already do).
  • Family tier — multiple kids, custom routines, and a parent dashboard.

Next: a parent setup mode, multi-child support, and an App Store wrapper.

What I learned building it

How I directed the AI

Start from the behavior, not the screens

I described the outcome — "a kid will actually want to finish their routine" — and let the mechanics follow: points instead of dollars (more game-like), progress that survives a refresh, puzzles gated behind completing the routine. Designing for the feeling first led to better features than a feature list would have.

What broke & how I fixed it

One backtick = white screen

A late version shipped a blank page. The culprit: nested template literals (backticks inside backticks) silently crashed the whole script. In a single-file app, one syntax slip takes everything down. The fix — rewrite with plain string concatenation, no nesting — and a habit of sanity-checking (line count, balance checks) after every big change.

The automation win

A backend-free "daily" engine

No server, but it still feels alive: a per-day storage key (ritual_YYYY-M-D) auto-resets progress each morning, and a day-of-year index rotates in a fresh puzzle every day. Real "daily app" behavior with zero infrastructure to run or pay for.

Tools that did the work

One file, iterated live

Pure HTML/CSS/JS with localStorage — no framework, no build step. I iterated version over version (points system → two puzzles → unlock animations) in fast passes, which is exactly where a single-file app shines: change, refresh, see it.
